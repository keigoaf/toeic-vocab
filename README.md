# TOEIC 単語トレーナー (PWA)

TOEICの単語・熟語を **間隔反復（Leitnerボックス方式）** で暗記するためのアプリです。
スマホのホーム画面に置いて、ネイティブアプリのように使えます（オフライン対応・端末内に進捗を永続保存）。

最終更新: 2026-07-23 01:21 (JST) / バージョン **1.2.0**

---

## 主な機能

- **間隔反復**: 正解した語は上のボックスへ進み、次の出題までの間隔が延びる。不正解ならBox1へ戻る。
- **出題周期の設定**: 各ボックスの「次回出題まで何日後か」を設定画面から自由に変更可能（初期値: 当日/1/3/7/14/30/60日）。
- **単語管理**: 追加・編集・削除・検索。単語には **品詞**（名詞・動詞など）を付けられ、めくった裏面に表示。
- **一括インポート / エクスポート**: テキストの貼り付けでまとめて登録・差し替え・バックアップ。
- **進捗の永続保存**: ブラウザの `localStorage` に保存。アプリを閉じても残る（同一URL内で保持）。

## スマホで「アプリ」として使う手順

ホーム画面に本物のアプリのように入れるには、**HTTPSで配信**する必要があります（Service Workerの制約）。
一番手軽なのは **GitHub Pages（無料・即HTTPS）** です。

### 1. GitHub Pages に配置する

**Git CLI の場合:**

```bash
cd toeic-vocab
git init
git add .
git commit -m "TOEIC vocab PWA v1.2.0"
git branch -M main
git remote add origin https://github.com/keigoaf/toeic-vocab.git   # ← 自分のリポジトリに
git push -u origin main
```

その後、GitHub のリポジトリで
`Settings → Pages → Build and deployment → Source: Deploy from a branch`
→ Branch を `main` / `/(root)` にして **Save**。

1分ほどで次のURLで公開されます（ユーザー名・リポジトリ名に合わせて読み替え）:

```
https://keigoaf.github.io/toeic-vocab/
```

### 2. ホーム画面に追加する

公開URLをスマホのブラウザで開き、ホーム画面に追加します。

- **iPhone (Safari)**: 共有ボタン → 「ホーム画面に追加」
- **Android (Chrome)**: メニュー(⋮) → 「アプリをインストール」または「ホーム画面に追加」

追加後はアイコンから全画面で起動し、オフラインでも動きます。

### （代替）自分のVPSで配信する場合

`168.144.41.110` 等の自前サーバでも配信できますが、Service Worker を有効にするには
**有効なHTTPS証明書**が必要です（自己署名不可）。Let's Encrypt などで証明書を用意し、
静的ファイルとして上記フォルダを配信してください。手軽さではGitHub Pagesが上です。

## 一括インポート / エクスポートのデータ形式

1行に1語。カンマ（または タブ）区切りで、**英語 , 意味 , 品詞/種別 , 例文** の順。
3列目・4列目は省略可能です。

- 3列目に **品詞**（名詞・動詞・形容詞・副詞・前置詞・接続詞・代名詞、または `n.` `v.` `adj.` 等）を書くと単語として登録。
- 3列目に **`熟語` または `idiom`** を書くと熟語として登録（品詞なし）。
- 空欄なら単語（品詞なし）として登録。

例:

```
acquire, 獲得する, 動詞, We plan to acquire a firm.
comply with, 順守する, 熟語
invoice, 請求書, 名詞
tentative, 仮の, adj.
```

エクスポートも同じ形式で出力されるので、そのままインポートに貼り戻せます（往復可能）。

## バックアップと注意点

- 進捗は `localStorage` に保存されます。これは **配信URL（ドメイン）ごと** に保持されます。
  配信先URLを変えると以前のデータは引き継がれません。
- 大事な単語リストは「単語」→「エクスポート」でテキストを控えておくと安全です。
  復元は「一括インポート」に貼り付けます。
- ブラウザの履歴・サイトデータを消すと `localStorage` も消えます。定期的なエクスポートを推奨。

## 開発メモ

- アプリ本体・SW・READMEには先頭に最終更新日時を記載しています。
  起動時にコンソールへ最終更新日時とバージョンを最初に出力し、画面右上にも表示します
  （どのバージョンを動かしているか後から確認できるように）。
- **アプリを更新したとき** は、`sw.js` 内の `CACHE` 名のバージョンも上げてください。
  古いキャッシュが確実に入れ替わります。
